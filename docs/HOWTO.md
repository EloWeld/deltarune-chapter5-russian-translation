# DELTARUNE глава 5 — русский перевод (руководство)

Проект живёт в `~/deltarune-ch5-ru/`. Перевод делается по чанкам (кускам) субагентами Claude Code — **с подписки, без API-ключа**.

## Файлы

| Что | Файл |
|---|---|
| Правила перевода (промпт для субагента) | `docs/PROMPT.md` |
| Источник целиком (англ. + яп. фолбэк), 17521 строка | `source_ch5.json` |
| Источник, нарезанный на 147 чанков по 120 строк | `chunks/chunk_000.json` … `chunk_146.json` |
| Переведённые чанки | `out/chunk_XXX.json` |
| Сборка + вшивание в игру | `scripts/build_install.py` |
| Пере-нарезка источника (если нужно) | `scripts/split_chunks.py` |
| Извлечение англ. из game.ios (уже сделано) | `scripts/extract_en_bytecode.py`, `scripts/build_source.py` |

**Важно:** чанки идут в порядке ключей движка (по объектам кода), а **не** в порядке прохождения. То есть «следующий чанк» ≠ «следующая сцена в игре». Но переводя по порядку 000→146, ты постепенно покрываешь весь текст. `chunk_000` — это вступление (создание Сосуда, самое начало главы).

## Как перевести очередные чанки (через субагента)

Для экономии **один субагент берёт сразу 9 чанков** (диапазон), а не один — меньше агентов, меньше накладных расходов. Попроси Claude: **«переведи chunks 010–016»** (или «следующие 9»). Claude запускает субагента с заданием вида:

> Прочитай правила `~/deltarune-ch5-ru/docs/PROMPT.md`. Затем по очереди обработай чанки chunk_010 … chunk_016: для каждого прочитай `~/deltarune-ch5-ru/chunks/chunk_NNN.json`, переведи КАЖДОЕ значение на русский строго по правилам (ключи не меняй, порядок сохрани, управляющие коды `^6 & % %% \M0 \E0 \cY \cW ~1 …` переноси дословно, `%` никогда не как часть слова) и запиши в `~/deltarune-ch5-ru/out/chunk_NNN.json` (валидный JSON, та же структура). В конце ответь, сколько чанков и строк перевёл.

Один субагент на 9 чанков ≈ 840 строк ≈ ~250–350k токенов подписки. Можно запускать несколько таких субагентов параллельно (например 6 агентов × 9 чанков = 54 чанка за заход).

**Модель:** субагентов перевода запускать на **Sonnet** (`model: "sonnet"` в вызове Agent). Без явного указания субагент наследует модель главного цикла (Opus) — для этой механической задачи это лишний расход. Держать одновременно не больше **7** работающих агентов, по мере завершения запускать следующие диапазоны.

## Проверка (валидация кодов)

После перевода Claude сверяет управляющие коды оригинала и перевода. Если где-то количество `^`, `&`, `%`, `\M…` не совпало — строка чинится (частая ошибка: `percent` → `%`, или лишняя пауза `^1`). Битые, но неисправленные строки в игре ломают текст, поэтому проверка обязательна.

## Вшить в игру

```
cd ~/deltarune-ch5-ru
python3 scripts/build_install.py
```

Скрипт собирает ВСЕ переведённые чанки + английский фолбэк на остальное и записывает в
`…/chapter5_mac/lang/lang_ja.json` (оригинал бэкапится в `lang_ja.json.orig.bak` один раз).

**Почему lang_ja.json:** глава 5 ванильная, её код грузит внешний текст только под японский язык (слота «русский» в коде ch5 нет, в отличие от модифицированных глав 1–4). Поэтому русский кладём на место японского, а в игре выбираем японский.

## Как увидеть перевод в игре

1. Запусти DELTARUNE, глава 5.
2. В настройках игры выбери язык **日本語 (японский)** — тогда игра подхватит наш `lang_ja.json`.
3. Переведённое покажется по-русски, ещё не переведённое — по-английски (это норма).

## Откат

```
cp "…/chapter5_mac/lang/lang_ja.json.orig.bak" "…/chapter5_mac/lang/lang_ja.json"
```

## Прогресс

Переведено: **все 147 чанков (000–146), 17521 строка**. Сводная валидация — `python3 scripts/validate_all.py`:
ключи покрывают `source_ch5.json` 1:1, дыр и дубликатов нет, расхождений по кодам нет
(единственное срабатывание — `scr_spellinfo_gml_109_0`, ложное: слэш в `Damage#w/ ICE`
это английское «w/», а не управляющий код).

Глоссарий унифицирован между чанками, перевод вшит в игру. Статус и принятый
канон имён — в `GLOSSARY_FIXES.md`.

После любой правки в `out/`: `python3 scripts/validate_all.py && python3 scripts/build_install.py`.
